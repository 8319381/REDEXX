БИРЖА ЛОГИСТИЧЕСКИХ СТАВОК
======================

Запуск приложения
----------------
1. Установка зависимостей:
   - В папке backend: npm install
   - В папке frontend: npm install

2. База данных (миграции):
   - Выполните SQL из backend/migrations/001_add_user_profile.sql
   - Выполните SQL из backend/migrations/002_create_requests_and_dictionaries.sql
   - Например: 
     psql -U postgres -d your_db -f backend/migrations/001_add_user_profile.sql
     psql -U postgres -d your_db -f backend/migrations/002_create_requests_and_dictionaries.sql

3. Запуск серверов:
   - Запуск backend: 
     cd backend
     npm start
     (сервер запустится на порту 3001)

   - Запуск frontend:
     cd frontend
     npm start
     (приложение откроется на http://localhost:3000)

4. Опционально: для проверки ИНН и валидации городов по DaData создайте в backend файл .env и добавьте:
   DADATA_API_KEY=ваш-ключ
   (ключ можно получить на https://dadata.ru/)
   Без ключа валидация городов будет выполняться через встроенный справочник в БД (таблица locations)

Работа с приложением
-------------------
1. Регистрация:
   - Выберите роль: заказчик, логист или администратор
   - Укажите имя и фамилию
   - Наименование компании и ИНН (проверка по DaData; в backend в .env задайте DADATA_API_KEY)
   - Email и пароль (не менее 8 символов: заглавные и строчные буквы, цифры, спецсимволы)
   - Подтверждение пароля

2. Возможности логиста (роль «Логист»):
   - Создание ставок по маршрутам
   - Просмотр встречных предложений от покупателей
   - Отслеживание изменений стоимости и сроков

3. Возможности заказчика (роль «Заказчик»):
   - Создание заявок на перевозку с указанием:
     * Дата готовности груза (обязательно)
     * Пункт отправки и назначения с валидацией через DaData (обязательно)
     * Тип контейнера: 20DC, 40DC, 40HC (обязательно)
     * Правила Инкотермс из справочника (обязательно)
     * Вес груза (опционально)
     * Объем, тип груза, желаемый срок поставки, вид транспорта, стоимость груза, комментарий (опционально)
   - Редактирование своих заявок
   - Удаление своих заявок
   - Просмотр всех ставок логистов
   - Создание встречных предложений
   - Отслеживание своих предложений по каждой ставке

Маршруты и транспорт
------------------
- Доступные маршруты: Москва – Пекин, Москва – Шанхай
- Тип транспорта: Ж/Д

Системные требования
------------------
- Node.js 14+ и npm
- Современный веб-браузер
- Порты 3000 и 3001 должны быть свободны
