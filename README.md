# Logistics Exchange

## Инструкция по запуску и развертыванию проекта

### Требования
- Node.js (версия 14 или выше)
- npm или yarn

### Локальный запуск

#### Бэкенд
1. Перейдите в директорию бэкенда:
   ```
   cd backend
   ```

2. Установите зависимости:
   ```
   npm install
   ```

3. Создайте файл `.env` с переменными окружения:
   ```
   PORT=3001
   JWT_SECRET=ваш_секретный_ключ
   ```

4. Запустите сервер:
   ```
   npm run dev
   ```
   Сервер запустится на порту 3001.

#### Фронтенд
1. Перейдите в директорию фронтенда:
   ```
   cd frontend
   ```

2. Установите зависимости:
   ```
   npm install
   ```

3. Запустите приложение:
   ```
   npm start
   ```
   Приложение запустится на порту 3000.

### Сборка для продакшена(хостинга)

#### Фронтенд
1. Перейдите в директорию фронтенда:
   ```
   cd frontend
   ```

2. Создайте оптимизированную сборку:
   ```
   npm run build
   ```
   Это создаст директорию `build` с оптимизированными файлами.

#### Размещение на хостинге

1. **Бэкенд**:
   - Разместите файлы бэкенда на сервере (например, VPS или облачном сервисе)
   - Установите зависимости: `npm install --production`
   - Настройте переменные окружения (PORT, JWT_SECRET)
   - Запустите сервер: `npm start` или используйте PM2: `pm2 start server.js`

2. **Фронтенд**:
   - Загрузите содержимое директории `build` на веб-сервер или используйте сервисы статического хостинга:
     - Netlify
     - Vercel
     - GitHub Pages
     - Firebase Hosting

3. **Настройка**:
   - Убедитесь, что в файлах фронтенда указан правильный URL API бэкенда
   - Настройте CORS на бэкенде для разрешения запросов с домена фронтенда
   - Настройте HTTPS для безопасного соединения

### Важные замечания
- Для продакшена замените хранение данных в памяти на настоящую базу данных
- Обеспечьте безопасность JWT_SECRET
- Настройте регулярное резервное копирование данных
